Let us first write down a general purpose R-function to perform Power Transformations

# General Purpose R function to perform power-transformation 
powerfun <- function(x, alpha) {
if(sum(x <= 0) > 0) stop("x must be positive")
if (alpha == 0)
  log(x)
else if (alpha > 0) {
  x^alpha
} else -x^alpha
}

Bump Rule #1

Let us use the data from Question 6 - murders.csv.

data <- read.csv("murders.csv", header = TRUE)
y <- data$total

We will vary the power on number of murders.

par(mfcol = c(1, 3))
a = seq(-1/2, 0, length.out=3)
for (i in 1:3) {
hist( powerfun( y + 1, a[i]), col=adjustcolor("grey", alpha = 0.5),
main= bquote(alpha == .(a[i])), xlab="", breaks=50 )
}

As you can see from the above graphs that the bump is on the higher value for \[ \alpha = -0.5 \] Using bump rule, when we increased the \(\alpha\) value to -0.25 and 0 subsequently, the histogram started to become more symmetric. We will continue the same until we feel that we have found the right \(\alpha\)

par(mfcol = c(1, 3))
b = seq(0, 1, length.out=3)
for (i in 1:3) {
hist( powerfun( y + 1, b[i]), col=adjustcolor("grey", alpha = 0.5),
main= bquote(alpha == .(b[i])), xlab="", breaks=50 )
}

Clearly, alpha seems to lie between 0 and 0.5 using bump rule.

par(mfcol = c(2, 3))

As you can see from above, the histogram seems most symmetric for \[ \alpha = 0.2 \] Hence using Bump Rule, we can pick our \(\alpha\) to be 0.2
